## Vì sao có sprint này

Design Figma (9 màn, xem `docs/design/figma/`) hoàn thiện sau khi Sprint B4 đóng. So sánh design với BE hiện tại lộ ra vài gap nhỏ; nếu để đến lúc làm FE mới phát hiện thì sẽ phải rework BE giữa sprint, phá vỡ contract "BE-first". B4.6 đóng các gap đó trước, để FE customer wire data 1:1 không bất ngờ.

Đây là **schema + service tweak**, không phải feature mới. Scope cố ý nhỏ.

## Vì sao pause B5

Sprint B5 (hardening + Railway deploy) là next stop ban đầu. Deploy BE đơn lẻ về lý thuyết đúng nhưng operationally tốn: mỗi lần FE phát hiện field thiếu / query sai / CORS host wrong → redeploy. Tốt hơn:

1. Xong B4.6 (sprint này) để API surface khớp design.
2. Build FE customer local, gọi `localhost:3000`.
3. Build FE admin local, gọi `localhost:3000`.
4. **Sau đó** Sprint B5 — deploy cả 3 repo cùng lúc thành 1 hệ thống đã validate.

Local-first work được vì Supabase/Stripe/Resend đã remote-hosted; trên `localhost` chỉ có Node process + 2 Next.js dev server.

## Sub-features

### B4.6.1 — Thêm `Tour.isFeatured`

```prisma
model Tour {
  // ...field cũ
  isFeatured  Boolean  @default(false) @map("is_featured")

  @@index([isFeatured, isPublished])  // query strip hero home
}
```

Drives:
- Strip "Trending Tour Packages" home page (3-card row trong màn Home Figma).
- Filter `?featured=true` trên `GET /tours` (B4.6.3).

Migration name: `add_tour_featured_flag`.

### B4.6.2 — Mở rộng enum `TourCategory`

```prisma
enum TourCategory {
  DAY
  PACKAGE
  CUSTOM
  HONEYMOON   // mới
  MUSICAL     // mới
}
```

Drives dropdown Services trong Figma nav (Honeymoon Packages / Tours Packages / Musical Events / Build Package). `Build Package` là flow riêng — xem BACKLOG.

Migration name: `extend_tour_category_enum`. Postgres enum extension không destructive (`ALTER TYPE ... ADD VALUE`), safe trên data hiện có.

### B4.6.3 — Query DTO: `featured` + sort enum

```ts
class ListToursQueryDto {
  // cũ: destination, category, minPrice, maxPrice, duration, page, limit
  featured?: boolean;
  sort?: 'createdDesc' | 'priceAsc' | 'priceDesc' | 'titleAsc' | 'titleDesc';
}
```

Map vào sort tabs trong Figma tour list ("Date / Price Low To High / Price High To Low / Name A-Z"). Sort mặc định vẫn `createdDesc`.

### B4.6.4 — Enrich card payload

Tour card trong Figma hiển thị:
- Average rating (★ 5.0)
- "120+ People" đang đi

`GET /tours` hiện chỉ trả raw tour row. Thêm vào mỗi card:

```ts
{
  // field cũ
  averageRating: number | null,   // null khi không có review approved
  reviewsCount: number,           // chỉ approved
  peopleGoing: number,            // tổng seat của các booking PAID trên các departure của tour
}
```

Implementation: mở rộng `findPublishedList` với aggregated subquery hoặc `$queryRaw` join. Index `reviews(tourId, isApproved)` đã có; cho `peopleGoing` sum `tour_departures.seats_booked` (đã được Stripe webhook + admin refund maintain).
